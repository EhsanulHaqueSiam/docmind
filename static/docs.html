API

All endpoints live under /api/v1. If API_KEY is set, include X-API-Key header in every request.

Query & Search

POST /api/v1/query Ask a question, get a cited answer ▾

Retrieves relevant chunks, then uses Gemini to generate a grounded answer with [filename] citations. Uses Flash by default; falls back to Pro for uncertain answers.

Request

FieldTypeDescription
questionstringYour question (1–10,000 chars, required)
use_probooleanForce Gemini Pro (default: false)

Response

FieldTypeDescription
answerstringGenerated answer with citations
sourcesstring[]Source filenames used
modelstring|nullModel used
chunks_usedintegerNumber of context chunks
fallbackbooleanWhether Pro fallback triggered
cURL
Python
JavaScript
bash
curl -X POST http://localhost:8000/api/v1/query \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the key findings?"}'

print("Sources:", data["sources"])

Workflows

Complete Examples

End-to-end code showing how to integrate DocMind into your applications.

Python: Full Workflow

python
"""DocMind: upload → search → query → delete."""
import httpx

BASE = "http://localhost:8000/api/v1"
client = httpx.Client(base_url=BASE, timeout=120)
# If API_KEY is set: client = httpx.Client(..., headers={"X-API-Key": "secret"})

# 1. Upload a document
with open("annual-report.pdf", "rb") as f:
    upload = client.post("/documents/upload", files={"file": f}).json()
print(f"Ingested: {upload['chunks']} chunks, ID: {upload['doc_id']}")

# 2. Search for relevant chunks (no LLM cost)
results = client.post("/search", json={"query": "revenue growth"}).json()
for r in results:
    print(f"  [{r['filename']}] score={r['score']:.3f}: {r['text'][:80]}...")

# 3. Ask a question (Gemini generates answer)
answer = client.post("/query", json={
    "question": "What was the company's revenue growth in 2024?",
}).json()
print(f"\n{answer['answer']}")
print(f"Sources: {answer['sources']}, Model: {answer['model']}")

# 4. Use Pro for complex questions
pro = client.post("/query", json={
    "question": "Compare Q1 vs Q4 across all metrics",
    "use_pro": True,
}).json()
print(pro["answer"])

# 5. List all documents
for doc in client.get("/documents").json():
    print(f"  {doc['filename']} — {doc['chunks']} chunks")

# 6. Check stats
stats = client.get("/stats").json()
print(f"Total points: {stats['points_count']}")

# 7. Delete
client.delete(f"/documents/{upload['doc_id']}")

JavaScript Client

javascript
class DocMind {
  constructor(base = "http://localhost:8000/api/v1", apiKey = null) {
    this.base = base;
    this.headers = { "Content-Type": "application/json" };
    if (apiKey) this.headers["X-API-Key"] = apiKey;
  }

  async query(question, usePro = false) {
    const r = await fetch(`${this.base}/query`, {
      method: "POST", headers: this.headers,
      body: JSON.stringify({ question, use_pro: usePro }),
    });
    return r.json();
  }

  async search(query, rerank = true) {
    const r = await fetch(`${this.base}/search`, {
      method: "POST", headers: this.headers,
      body: JSON.stringify({ query, rerank }),
    });
    return r.json();
  }

  async upload(file) {
    const form = new FormData();
    form.append("file", file);
    const h = this.headers["X-API-Key"]
      ? { "X-API-Key": this.headers["X-API-Key"] } : {};
    const r = await fetch(`${this.base}/documents/upload`, {
      method: "POST", headers: h, body: form,
    });
    return r.json();
  }

  async documents(offset = 0, limit = 50) {
    return (await fetch(`${this.base}/documents?offset=${offset}&limit=${limit}`,
      { headers: this.headers })).json();
  }

  async delete(docId) {
    return (await fetch(`${this.base}/documents/${docId}`,
      { method: "DELETE", headers: this.headers })).json();
  }
}

// Usage
const dm = new DocMind("http://localhost:8000/api/v1", "your-api-key");
const answer = await dm.query("Summarize the main points");
const chunks = await dm.search("neural networks", false);

# Authenticated request

Internals

Architecture

How DocMind processes documents and answers questions.

Query Pipeline

POST /api/v1/query │ ├─ Embed query gemini-embedding-001, task=RETRIEVAL_QUERY ├─ Vector search Qdrant, top_k=10 candidates ├─ Rerank Gemini Flash scores relevance → top 5 ├─ Generate Gemini Flash builds answer from context ├─ Fallback? If uncertain → retry with Gemini Pro └─ Response answer + sources + metadata

Ingestion Pipeline

Upload / File watcher │ ├─ Validate extension, size, filename sanitization ├─ Deduplicate SHA-256 content hash → skip if exists ├─ Parse Docling → structured document model ├─ Chunk HierarchicalChunker → semantic segments ├─ Embed gemini-embedding-001, task=RETRIEVAL_DOCUMENT └─ Index Qdrant upsert with metadata + headings

Design Decisions

DecisionWhy
Asymmetric embeddingsDifferent task types for docs vs queries improves retrieval accuracy
Flash → Pro fallbackCheap model first; expensive only when Flash is uncertain
Content-hash dedupSHA-256 prevents re-ingesting identical files
Gemini rerankingLLM reranks vector results for better relevance
Exponential backoffAll API calls retry on transient failures
File watcherDrop files in docs/ → auto-ingested with debounce
Production

Deployment

Three options depending on your needs.

Local

Free — Docker only

Development and testing. Qdrant runs as a Docker container alongside the app.

VPS / Server

$5–20/mo

Self-hosted production. Any Linux server with Docker. Use a service account for GCP auth.

Cloud Run

Pay per request

Fully managed, scales to zero. Requires Qdrant Cloud and GCS bucket.

Create a GCP Service Account

bash
export PROJECT_ID=my-docmind

# Create service account
gcloud iam service-accounts create docmind-sa \
  --display-name="DocMind"

# Grant Vertex AI access
gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:docmind-sa@$PROJECT_ID.iam.gserviceaccount.com" \
  --role="roles/aiplatform.user"

# Download key
gcloud iam service-accounts keys create sa-key.json \
  --iam-account="docmind-sa@$PROJECT_ID.iam.gserviceaccount.com"

# Copy to VPS
scp sa-key.json user@your-server:~/docmind/

Google Cloud Run

Serverless deployment. Requires Qdrant Cloud (free tier: 1 GB) and a GCS bucket.

bash
export PROJECT_ID=my-docmind

# 1. Create GCS bucket for documents
gsutil mb -l us-central1 gs://$PROJECT_ID-docs

# 2. Store secrets
echo -n "qdrant-api-key" | gcloud secrets create QDRANT_API_KEY --data-file=-
echo -n "your-api-key" | gcloud secrets create DOCMIND_API_KEY --data-file=-

# 3. Build and push image
gcloud builds submit --tag gcr.io/$PROJECT_ID/docmind-api:latest

# 4. Deploy
sed -i "s/PROJECT_ID/$PROJECT_ID/g" deploy/cloudrun.yaml
gcloud run services replace deploy/cloudrun.yaml --region us-central1

# 5. Allow public access
gcloud run services add-iam-policy-binding docmind-api \
  --region=us-central1 --member="allUsers" --role="roles/run.invoker"
