Welcome to DocMind

Document intelligence API powered by Gemini, Docling, and Qdrant. Upload documents, ask questions, get grounded answers with citations.

Gemini 2.5 Qdrant Vector DB Docling Parser 13+ File Formats

#Getting Started

Get DocMind running locally in under 5 minutes.

What You Need (Accounts & Services)

DocMind requires two external services. Here's exactly what to set up:

ServiceCostWhat ForHow to Get
Google Cloud (GCP) Free tier available Gemini models for embeddings, reranking, and answer generation Create a project at console.cloud.google.com
Qdrant Free (self-hosted) Vector database for storing and searching document embeddings Run via Docker (included) or use Qdrant Cloud

bash

# Linux:

gcloud services enable aiplatform.googleapis.com

gcloud auth application-default login

#API Reference

All endpoints are under /api/v1. If API_KEY is set, include X-API-Key header.

Query

POST /api/v1/query Ask a question, get a grounded answer ▾

Retrieves relevant document chunks, then uses Gemini to generate a grounded answer with source citations. Uses Flash by default with automatic Pro fallback for uncertain answers.

Request Body

FieldTypeDescription
questionstringThe question to answer (1–10,000 chars, required)
use_probooleanForce Gemini Pro model (default: false)

Response

FieldTypeDescription
answerstringGenerated answer with [filename] citations
sourcesstring[]List of source filenames used
modelstring | nullModel used (e.g. "gemini-2.5-flash")
chunks_usedintegerNumber of context chunks used
fallbackbooleanWhether Pro fallback was triggered
cURL
Python
JavaScript
bash
curl -X POST http://localhost:8000/api/v1/query \
  -H "Content-Type: application/json" \
  -d '{"question": "What are the key findings in the report?"}'

import httpx

resp = httpx.post("http://localhost:8000/api/v1/query", json={
    "question": "What are the key findings in the report?",
    "use_pro": False,
})

print("Sources:", data["sources"])

Performs semantic vector search and optional Gemini reranking. Returns raw document chunks without generating an LLM answer. Useful for search UIs, debugging, or building custom pipelines.

#Complete Examples

End-to-end workflows showing how to use DocMind in your applications.

Python: Upload, Query, and Cleanup

python
"""Complete DocMind workflow: upload → query → search → delete."""
import httpx

BASE = "http://localhost:8000/api/v1"
# If API_KEY is set, uncomment:
# HEADERS = {"X-API-Key": "your-secret-key"}
HEADERS = {}

client = httpx.Client(base_url=BASE, headers=HEADERS, timeout=120)

# 1. Check health
health = client.get("/health").json()
print(f"Status: {health['status']}, Qdrant: {health['qdrant']}")

# 2. Upload a document
with open("annual-report.pdf", "rb") as f:
    resp = client.post("/documents/upload", files={"file": f})
    upload = resp.json()
    print(f"Uploaded: {upload['filename']} → {upload['chunks']} chunks")
    doc_id = upload["doc_id"]

# 3. List all documents
docs = client.get("/documents").json()
for doc in docs:
    print(f"  {doc['filename']} ({doc['chunks']} chunks)")

# 4. Search for relevant chunks (no LLM generation)
results = client.post("/search", json={"query": "revenue growth 2024"}).json()
print(f"\nFound {len(results)} relevant chunks:")
for r in results:
    print(f"  [{r['filename']}] score={r['score']:.3f}: {r['text'][:80]}...")

# 5. Ask a question (with LLM-generated answer)
answer = client.post("/query", json={
    "question": "What was the company's revenue growth in 2024?",
}).json()
print(f"\nAnswer ({answer['model']}, {answer['chunks_used']} chunks):")
print(answer["answer"])
print(f"Sources: {', '.join(answer['sources'])}")
print(f"Fallback to Pro: {answer['fallback']}")

# 6. Use Gemini Pro for complex questions
pro_answer = client.post("/query", json={
    "question": "Compare Q1 and Q4 performance across all metrics",
    "use_pro": True,
}).json()
print(f"\nPro answer ({pro_answer['model']}):")
print(pro_answer["answer"])

# 7. Check stats
stats = client.get("/stats").json()
print(f"\nCollection: {stats['points_count']} points, status={stats['status']}")

# 8. Delete the document
client.delete(f"/documents/{doc_id}")
print(f"\nDeleted document {doc_id}")

JavaScript: Build a Search Interface

javascript
/**
 * DocMind client for browser or Node.js applications.
 */
class DocMindClient {
  constructor(baseUrl = "http://localhost:8000/api/v1", apiKey = null) {
    this.baseUrl = baseUrl;
    this.headers = { "Content-Type": "application/json" };
    if (apiKey) this.headers["X-API-Key"] = apiKey;
  }

  async query(question, usePro = false) {
    const resp = await fetch(`${this.baseUrl}/query`, {
      method: "POST",
      headers: this.headers,
      body: JSON.stringify({ question, use_pro: usePro }),
    });
    if (!resp.ok) throw new Error(`Query failed: ${resp.status}`);
    return resp.json();
  }

  async search(query, rerank = true) {
    const resp = await fetch(`${this.baseUrl}/search`, {
      method: "POST",
      headers: this.headers,
      body: JSON.stringify({ query, rerank }),
    });
    if (!resp.ok) throw new Error(`Search failed: ${resp.status}`);
    return resp.json();
  }

  async upload(file) {
    const formData = new FormData();
    formData.append("file", file);
    const headers = {};
    if (this.headers["X-API-Key"]) headers["X-API-Key"] = this.headers["X-API-Key"];
    const resp = await fetch(`${this.baseUrl}/documents/upload`, {
      method: "POST", headers, body: formData,
    });
    if (!resp.ok) throw new Error(`Upload failed: ${resp.status}`);
    return resp.json();
  }

  async listDocuments(offset = 0, limit = 50) {
    const resp = await fetch(
      `${this.baseUrl}/documents?offset=${offset}&limit=${limit}`,
      { headers: this.headers },
    );
    return resp.json();
  }

  async deleteDocument(docId) {
    const resp = await fetch(`${this.baseUrl}/documents/${docId}`, {
      method: "DELETE", headers: this.headers,
    });
    if (!resp.ok) throw new Error(`Delete failed: ${resp.status}`);
    return resp.json();
  }

  async stats() {
    const resp = await fetch(`${this.baseUrl}/stats`, { headers: this.headers });
    return resp.json();
  }

  async health() {
    const resp = await fetch(`${this.baseUrl}/health`, { headers: this.headers });
    return resp.json();
  }
}

// Usage:
const dm = new DocMindClient("http://localhost:8000/api/v1");
const results = await dm.search("machine learning techniques");
const answer = await dm.query("Explain the ML techniques used in the paper");

#Architecture

How DocMind processes documents and answers questions.

Query Pipeline

POST /api/v1/query │ ├─ Embed query (gemini-embedding-001, task=RETRIEVAL_QUERY) ├─ Vector search (Qdrant, top_k=10 candidates) ├─ Rerank (Gemini Flash, top_k=5 passages) ├─ Generate answer (Gemini Flash with context) ├─ Fallback check (if answer uncertain → retry with Gemini Pro) └─ Return answer + sources + metadata

Ingestion Pipeline

POST /api/v1/documents/upload or File watcher (docs/) │ ├─ Validate (extension, size, filename sanitization) ├─ Deduplicate (SHA-256 content hash → skip if exists) ├─ Parse (Docling → structured document) ├─ Chunk (HierarchicalChunker → semantic chunks) ├─ Embed (gemini-embedding-001, task=RETRIEVAL_DOCUMENT) └─ Index (Qdrant upsert with metadata)

Key Design Decisions

DecisionRationale
Asymmetric embeddingsDifferent task types for documents vs queries improves retrieval accuracy
Flash → Pro fallbackCheap fast model first; expensive model only for uncertain answers
Content-hash dedupSHA-256 prevents re-ingesting identical files, saving API costs
Gemini rerankingLLM reranks vector results for better relevance ordering
Exponential backoffAll Gemini and Qdrant calls retry on transient failures
File watcherDrop files in docs/ — auto-ingested with debounced processing

Using a GCP Service Account on VPS

For production VPS deployments, create a service account instead of using personal credentials:

bash
# On your local machine:
export PROJECT_ID=my-docmind-project

# Create service account
gcloud iam service-accounts create docmind-sa \
  --display-name="DocMind Service Account"

# Grant Vertex AI permissions
gcloud projects add-iam-policy-binding $PROJECT_ID \
  --member="serviceAccount:docmind-sa@$PROJECT_ID.iam.gserviceaccount.com" \
  --role="roles/aiplatform.user"

# Download key file
gcloud iam service-accounts keys create docmind-sa-key.json \
  --iam-account="docmind-sa@$PROJECT_ID.iam.gserviceaccount.com"

# Copy to your VPS
scp docmind-sa-key.json user@your-server-ip:~/docmind/

Then in your docker-compose.yml, add the credentials volume:

yaml
  docmind:
    build: .
    volumes:
      - ./docs:/app/docs
      - ./docmind-sa-key.json:/app/service-account.json:ro
    environment:
      - GOOGLE_APPLICATION_CREDENTIALS=/app/service-account.json
